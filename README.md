# OIDC Identity & Authorization Control Center

A secure, high-performance, developer-focused OpenID Connect (OIDC) and OAuth 2.0 compliant identity provider engine.

This project is an RFC-compliant OpenID Connect (OIDC) and OAuth 2.0 identity provider engine built with Express, PostgreSQL, and Drizzle ORM.
The architecture enforces strict Proof Key for Code Exchange (PKCE) flow validations, asymmetric RS256 token signing, and HTTP-only session cookies.
It features a strict monochrome React SPA developer console with client self-registration, interactive scope consent screens, and admin approval queues.

---

## Key Features & Specifications

* **Identity & Authentication**: Supports user registration, sign-in, and persistent session state tracking.
* **Authorization Grant**: Implementation of the Authorization Code Grant Flow supporting `state`, `nonce`, and mandatory `code_challenge` (PKCE) parameters.
* **Token Service**: Issues cryptographically signed JSON Web Tokens (JWT) using the asymmetric **RS256** algorithm.
* **Scope Consent Guard**: Middleware intercepts auth codes until the user explicitly grants scope authorizations.
* **JWKS Discovery**: Public metadata endpoints (`/.well-known/openid-configuration` and `/.well-known/jwks.json`) serve JWK structures for public key verification.
* **Admin Approval Queue**: Client applications must register callback redirect URIs and description manifests, which administrators approve to generate live credentials.

## Tech Stack

* **Backend**: Node.js, Express, TypeScript, `tsc-watch`
* **Database & ORM**: PostgreSQL (running in Docker Compose), Drizzle ORM
* **Frontend**: React 19, Vite 8, TypeScript, Lucide React

## Core Endpoint Spec

### OIDC Standards
* `GET  /.well-known/openid-configuration` - OpenID metadata listing
* `GET  /.well-known/jwks.json` - Asymmetric public verification keys
* `GET  /o/authenticate` - User authentication landing SPA page
* `POST /o/authenticate/sign-in` - Session login validation handler
* `POST /o/authenticate/sign-up` - Account creation endpoint
* `GET  /o/authorize` - Authorization code request handler
* `GET  /o/authorize/consent` - Scope decision SPA view
* `POST /o/authorize/decision` - Scope authorization handler
* `POST /o/token` - Token exchange endpoint (code to JWT tokens)
* `GET  /o/userinfo` - Token-authorized user claims resolver
* `POST /o/logout` - User session termination

### Admin & Client Provisioning
* `GET  /clients/register` - Application registry registration form SPA view
* `POST /clients/register` - Registers a client request to queue
* `GET  /admin/dashboard` - Admin approvals dashboard SPA view
* `GET  /admin/registrations` - Lists pending client registrations
* `POST /admin/registrations/:id/approve` - Authorizes client app generation
* `POST /admin/registrations/:id/reject` - Rejects client registration

## How to Get Started

### Quick Start
To build the frontend, run database migrations, and spin up both dev servers concurrently:
```bash
# Using the NPM start script:
pnpm start

# Or using the platform wrappers:
./run.sh    # macOS / Linux
run.bat     # Windows
```

### Manual Dev Setup
If you want to run steps individually:
1. Spin up the PostgreSQL database container:
   ```bash
   docker compose up -d
   ```
2. Run database migrations:
   ```bash
   pnpm run db:generate
   pnpm run db:migrate
   ```
3. Run the hot-reloading development servers (Backend: 8080, Frontend: 5173):
   ```bash
   pnpm run dev:all
   ```
